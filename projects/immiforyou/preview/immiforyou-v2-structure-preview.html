国家详情页骨架

ContentLayout

国家页以后仍然是最稳、最能承重的一类页面。主体内容尽量静态，承担长期搜索、长期信任、长期教育；动态部分集中在“相关新闻聚合层”，而且只承接站内原创政策文章。

Hero / 概览 国家定位、适合人群、核心判断、官方来源可信感。
路径拆解 / 要求 / 成本周期 这些都是 evergreen 主体，默认静态优先，靠周期性复核更新，而不是频繁重写。
适配人群 / FAQ / 官方链接 继续服务“读懂”和“做判断”，而不是只给碎片结论。
区块 更新方式 角色
路径说明 静态优先 承接长期搜索与长期理解
要求 / 成本 / 周期 静态优先 + 周期复核 降低长期维护负担
相关新闻聚合 动态 只承接站内原创政策文章，补时效性
这里最关键的结构判断是：国家页不是每天都重写，而是把“活”的部分集中在站内原创新闻聚合层。

自动聚合新闻层

Only Internal Original News

这块不是外部抓取，也不是新的原创池。它只从你网站自己已经发布的原创政策文章里，挑出和当前国家相关的内容，显示摘要并链接回全文。

这里会显示摘要，而不是把新闻正文重新复制一遍。作用是给国家页提供动态时效感，并把用户拉回原创文章全文。

如果文章和当前国家有关，也可以进入这里的相关动态层。核心原则是：国家页动态内容只来自站内原创。

新闻文章页骨架

Long-Form Authority Template

新闻文章页以后不只是政策快讯，而是深度长文入口。作者信息要清楚，结构要清楚，来源要清楚，正文要能承住表格、FAQ、persona 影响分析。

作者区块 作者中文名、英文名、负责区域。不是装饰，而是责任归属与可信度的一部分。
深正文区块 按 `D61` 深内容规则设计，不走短资讯样式。
来源 + FAQ + 相关推荐 让文章既能承接搜索，也能承接信任和站内跳转。

诊断页骨架

UtilityLayout

诊断页不是普通工具页，它继续保留核心服务入口地位。DeepSeek 路径诊断能力在这里，前端负责承接体验，真实判断逻辑继续留在受保护的 Worker 边界后面。

前段 信任说明、资格判断、问卷壳层。
中段 加载状态、结果壳层、解释层。
边界 支付和更完整商业化可以后置，但服务入口地位不变。
这个预览不是最终设计稿，只是把 `v2` 结构翻译成你可以直接看的网页轮廓。
