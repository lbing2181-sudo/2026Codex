# Character Continuity System

## Objective

Guarantee that recurring characters inside the same universe remain visibly and narratively consistent across future prompts, posts, and AI handoffs.

## Rule

Once a recurring character is promoted into canon, she is no longer a loose archetype.
She becomes a locked asset.

## What Must Stay Consistent

1. face structure
2. age band
3. body proportions
4. carriage and posture grammar
5. hair logic
6. signature adornments
7. wardrobe silhouette
8. color anchors
9. rank and social function
10. emotional register
11. symbolic objects
12. naming and title

## Required Dossier Fields

Each recurring woman should eventually have a dossier containing:

- dossier ID
- full name
- short title
- realm
- rank
- age band
- facial markers
- body presence
- hair description
- eye description
- skin tone language
- signature silhouette
- signature jewelry
- signature fabrics
- signature colors
- signature symbol
- default emotional temperature
- desire profile
- danger profile
- approved prompt anchors
- forbidden drift notes

## Drift Types To Prevent

- face drift
- body drift
- costume drift
- palette drift
- temperament drift
- hierarchy drift
- civilization drift

## Prompt Rule

If a recurring character appears, the prompt must reference her locked dossier traits before adding scene-specific variables.

## Publishing Rule

Do not publish two visibly incompatible versions of the same recurring character.
If continuity breaks, regenerate before posting.

## Handoff Rule

Future AIs should read this file and the character dossiers before generating any recurring figure.
