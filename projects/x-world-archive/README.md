# X World Archive

## Role In 2026Codex

This folder is the control-layer project entry for a premium X-native virtual worldbuilding account.
Its job is to turn AI image generation, lore writing, and account publishing into one coherent creator system.

## Core Thesis

- Build a work, not a feed.
- Build a universe, not a gallery.
- Build a recognizable archive, not a pile of prompts.
- Use sensuality, danger, and longing as narrative energy, not as cheap exposure.

## Operating Rule

- The public account must feel like one supreme universe with many sealed archives inside it.
- The first release phase must stay narrow and memorable.
- Hidden realms may be prepared early but not released early.
- Durable memory belongs in repository files, not in chat.

## Folder Map

- `CONSTITUTION.md`: non-negotiable rules
- `BRAND_BIBLE.md`: master identity and aesthetic
- `WORLD_BIBLE.md`: universe structure and realm strategy
- `CHARACTER_CONTINUITY_SYSTEM.md`: recurring character locking rules
- `X_ACCOUNT_OS.md`: account setup and publishing system
- `ACCOUNT_IDENTITY_DRAFT.md`: handle, display name, and bio drafts
- `VOICE_GUIDE.md`: English writing standards
- `PROMPT_ARCHITECTURE.md`: reusable prompt engine
- `AVATAR_SYSTEM.md`: avatar and profile identity system
- `PIPELINE.md`: creation and publishing workflow
- `DECISIONS.md`: project decisions
- `WORKING_MEMORY.md`: live operating state
- `seasons/`: season plans
- `characters/`: character dossiers
- `lore/`: lore fragments and world notes
- `posts/`: post systems and publishing assets
- `assets/`: profile, visual, and symbolic asset direction
