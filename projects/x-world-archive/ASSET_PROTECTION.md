# Asset Protection

## Objective

Protect the universe, brand, characters, images, and release assets of `Sable Archive` through layered digital-asset discipline rather than wishful thinking.

## Core Principle

Do not rely on any single protection method.

The protection system must combine:

- brand identity
- evidence chain
- release metadata
- fingerprinting
- platform enforcement
- durable internal records

## What We Can Protect Well

- brand name
- emblem/logo
- exact images
- exact captions and articles
- character dossiers
- universe documents
- release sequencing and archive numbering

## What We Can Protect Partially

- recurring character combinations
- repeated visual systems
- serialized world structures
- distinctive canon packages

## What We Cannot Fully Lock Down

- broad themes
- abstract style
- general mood
- people being influenced by the work without direct copying

## Protection Layers

### 1. Brand Layer

Use one stable brand identity everywhere:

- `Sable Archive`
- one emblem
- one archive logic
- one profile system

This increases confusion-based enforcement power when copycats imitate too closely.

### 2. Evidence Layer

Every important asset should have a verifiable creation trail.

Store:

- prompt versions
- references used
- draft outputs
- selected master file
- editing notes
- timestamps
- publication URL
- file hash
- associated dossier IDs

### 3. Credential Layer

When possible, use provenance-friendly export and content credentials workflows.

This is useful as a supporting authenticity layer, but it is not sufficient on its own because some platforms and repost paths strip metadata.

### 4. Fingerprint Layer

Use invisible or semi-invisible release markers as auxiliary evidence.

Examples:

- release-version file hashing
- subtle hidden fingerprint variants
- internal release signatures tied to asset IDs

These should support evidence, not replace it.

### 5. Platform Enforcement Layer

Prepare for:

- copyright reporting
- impersonation reporting
- repeat infringement logging

The value comes from speed and completeness of reporting.

### 6. Legal Layer

Increase human authorship in the chain through:

- selection
- editing
- arrangement
- naming
- dossier creation
- article writing
- release sequencing

The stronger the human-authored system, the stronger the defensible asset package.

## Asset Passport Standard

Each major release asset should eventually receive:

- asset ID
- universe ID
- season ID
- realm ID
- archive class
- character ID if applicable
- prompt version
- master file path
- public release path
- master hash
- public hash
- release date
- publication URL
- credential status
- fingerprint status
- enforcement notes

## Master Rule

If an asset matters, it must be traceable.
If it is not traceable, it is weakly protected.
